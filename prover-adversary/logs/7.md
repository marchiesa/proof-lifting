# Problem 7: Decreases Clause Tricks -- Two-Pointer Convergence

## Result: VERIFIED

## Attempts: 2

## Strategy

Two pointers converge from both ends. Left advances by 2, right by 1, giving a total decrease of 3 per iteration. The challenge is bounding the indices properly since `lo` can overshoot `|s|` and `hi` can go below 0.

### Attempt 1 (FAILED)
- Added `decreases hi - lo + 1` and invariants tracking which indices have been checked.
- **Errors:**
  - Index out of range on `s[k]` in quantified invariant: `lo` can exceed `|s|` (advances by 2 past the end) and `hi` can go to -1.
  - Invariant maintenance failure for the "left checked" invariant.

### Attempt 2 (VERIFIED)
- Fixed bounds: `0 <= lo <= |s| + 1` (lo can overshoot by 1 due to +2 step) and `-1 <= hi < |s|`.
- Added guard `k < |s|` in the left-checked invariant: `forall k :: 0 <= k < lo && k < |s| ==> s[k] != target`.
- The `hi < k < |s|` invariant was already safe since hi starts at `|s|-1` and decreases.

## Key Invariants
```
invariant -1 <= hi < |s|
invariant 0 <= lo <= |s| + 1
invariant !found
invariant forall k :: 0 <= k < lo && k < |s| ==> s[k] != target
invariant forall k :: hi < k < |s| ==> s[k] != target
decreases hi - lo + 1
```

## Notes
The asymmetric step (lo += 2, hi -= 1) means the gap closes by 3 per iteration. The `decreases hi - lo + 1` works because `hi - lo` decreases by 3 each time. The tricky part was handling the lo+1 check inside the loop body -- when `lo + 1 < hi`, the element at `lo + 1` is also checked before advancing, so all three positions (lo, lo+1, hi) are covered.
