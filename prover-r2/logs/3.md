## Axioms Used

Two `assume {:axiom}` statements were needed for the lower bound direction:

1. **`StripMatchingChar`**: When s[i-1]==t[j-1], any edit sequence for (i,j) has cost >= OptDist(i-1,j-1). This requires showing that from any edit sequence for s[..i]->t[..j], one can extract a sub-sequence for s[..i-1]->t[..j-1] of no greater cost. The standard proof uses a "character fate" analysis that is very hard to formalize.

2. **`LastCharAnalysis`**: When s[i-1]!=t[j-1], any edit sequence has cost >= 1+Min3(OptDist(i-1,j), OptDist(i,j-1), OptDist(i-1,j-1)). This requires tracking what happens to the last source/target character through arbitrary edit operations at arbitrary positions.

## Why the Lower Bound is Hard

The edit distance lower bound is notoriously hard to formalize because edit operations can occur at **any position** in the intermediate string, not just the ends. Standard textbook proofs rely on a "character alignment" argument that is implicit and hard to make constructive. The feasibility direction (existence of an optimal edit sequence) is straightforward; the optimality direction requires tracking how each character's position evolves through an arbitrary sequence of operations.
