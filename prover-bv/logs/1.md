# Problem 1: Popcount (count set bits)

## Result: VERIFIED (attempt 2 of 10)

## Problem
Count the number of 1-bits in a bv8 using the `x & (x-1)` trick (clears lowest set bit).
Spec uses a recursive ghost function PopcountSpec that counts bits from LSB via shifts.

## Why Z3 struggles
The recursive PopcountSpec operates bit-by-bit from LSB, but the loop uses the algebraic trick `x & (x-1)`. Z3 cannot automatically connect the bit-clearing trick to the recursive definition without help.

## Solution approach

### Attempt 1 (FAILED)
Added a lemma `PopcountClearLowest(x)` with postconditions:
- `PopcountSpec(x) > 0` (when x != 0)
- `PopcountSpec(x & (x-1)) == PopcountSpec(x) - 1`

With an empty body, Z3 could not prove either postcondition -- the recursive function was not unrolled deeply enough.

### Attempt 2 (SUCCESS)
Added `{:fuel PopcountSpec, 9}` attribute to the lemma. This tells Z3 to unroll PopcountSpec up to 9 times (one more than the 8 bits), which is enough for Z3 to fully evaluate the recursive function on any concrete bv8 value and verify the property.

The loop invariant is: `count + PopcountSpec(tmp) == PopcountSpec(x)`.
The lemma is called at each iteration to justify that the invariant is preserved when `tmp` is updated to `tmp & (tmp - 1)`.
