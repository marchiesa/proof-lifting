# Problem 3: Bit reversal

## Result: VERIFIED (attempt 1 of 10)

## Problem
Reverse all 8 bits of a bv8: bit i of output equals bit (7-i) of input.
The loop shifts bits from input LSB to output LSB, building the reversed value.

## Why Z3 struggles
The postcondition is a universal quantifier over bit positions. Each loop iteration affects ALL bits of r (via the shift), and Z3 must reason about bitvector shifts combined with quantifier instantiation.

## Solution approach

Introduced a ghost function `PartialReverse(x, j)` that computes the expected value of r after j iterations of the loop:
```
PartialReverse(x, 0) = 0
PartialReverse(x, j) = (PartialReverse(x, j-1) << 1) | GetBit(x, j-1)
```

This mirrors exactly what the loop does: shift r left by 1, then OR in the next bit from x.

Two lemmas with `{:fuel ..., 9}` to allow full unrolling over all 8 bits:

1. `PartialReverseCorrect`: After j iterations, the bits of PartialReverse(x, j) at positions [0, j) are the reversed bits from x, and positions [j, 8) are zero.

2. `PartialReverseStep`: Proves that `(r << 1) | (tmp & 1) == PartialReverse(x, j+1)` given the loop invariants.

The loop invariant tracks `r == PartialReverse(x, j)` and `tmp == x >> j`. At loop exit with j==8, we invoke `PartialReverseCorrect(x, 8)` to establish the postcondition.
