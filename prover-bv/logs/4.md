# Problem 4: Count leading zeros

## Result: VERIFIED (attempt 1 of 10)

## Problem
Count the number of leading (most significant) zero bits in a bv8.
The implementation scans from MSB (position 7) downward. The spec uses recursive `ClzSpec(x, pos)` that counts from the top bit.

## Why Z3 struggles
The loop scans bits from position 7 down to 0, while the recursive spec processes top to bottom. Relating the loop's partial scan to the recursive spec requires understanding how ClzSpec unrolls, combined with bitvector shift operations and quantifier/trigger issues.

## Solution approach

Key insight: the loop invariant is `count + ClzSpec(x, pos + 1) == Clz(x)`.

- At loop entry (pos=7, count=0): `0 + ClzSpec(x, 8) == Clz(x)` -- trivially true.
- Each iteration: if bit at pos is 0, ClzSpec(x, pos+1) = 1 + ClzSpec(x, pos), so incrementing count and decrementing pos preserves the invariant.
- At exit with bit at pos == 1: ClzSpec(x, pos+1) = 0, so count == Clz(x).
- At exit with pos == -1: ClzSpec(x, 0) = 0, so count == Clz(x).

Added a helper lemma `ClzUnroll` with `{:fuel ClzSpec, 2}` that proves the single unrolling step: when bit at pos is 0, `ClzSpec(x, pos+1) == 1 + ClzSpec(x, pos)`. This is called inside the loop body before updating count and pos.
