# Problem 5: Producer-Consumer Bounded Buffer

## Result: VERIFIED (4 attempts)

## Approach

### Key Invariant
The central insight is that `output + buffer == input[..prodIdx]` at every step:
- `output == input[..consIdx]` (consumed prefix)
- `buffer == input[consIdx..prodIdx]` (pending items)
- `prodIdx == CountTrue(schedule[..step])` (items produced so far)
- `consIdx == CountFalse(schedule[..step])` (items consumed so far)

### Helper Lemmas

1. **`CountTrueStep`/`CountFalseStep`**: `CountTrue(s[..n+1]) == CountTrue(s[..n]) + (if s[n] then 1 else 0)`. Proved by induction on n, with careful handling of the sequence decomposition `s[..n] == [s[0]] + s[1..n]`.

2. **`CountTruePlusFalse`**: `CountTrue(s) + CountFalse(s) == |s|`. Used to establish `prodIdx + consIdx == step`.

3. **`CountTrueSplit`**: `CountTrue(s) == CountTrue(s[..n]) + CountTrue(s[n..])`. Used to show that `CountTrue(schedule[..step]) < CountTrue(schedule)` when `schedule[step]` is true (i.e., there are still items to produce).

4. **`CountTrueFirstTrue`** + **`CountTrueNonNeg`**: If `s[0]` is true, then `CountTrue(s) >= 1`. Needed to establish `CountTrueNonNeg(s[1..])` first since Dafny couldn't automatically deduce that `CountTrue(s[1..]) >= 0`.

### Errors Encountered

1. **Attempt 1**: Sequence bounds errors in `CountTrueSplit`/`CountFalseSplit` when `n == |s|` (off by one). Also, buffer invariant bounds errors when indices weren't established yet.
2. **Attempt 2**: `CountTrue(schedule[step..]) >= 1` when `schedule[step]` is true couldn't be proved directly.
3. **Attempt 3**: `CountTrueFirstTrue` lemma postcondition failed because Dafny needed explicit non-negativity of `CountTrue(s[1..])`.
4. **Attempt 4**: Added `CountTrueNonNeg` call. Verified.

### Why the FIFO Property Works
The buffer acts as a queue: items are appended in order (produce) and removed from the front (consume). The invariant `buffer == input[consIdx..prodIdx]` directly establishes that `output[k] == input[k]` for all consumed items, which gives `output == input` at the end when all items have been consumed.
