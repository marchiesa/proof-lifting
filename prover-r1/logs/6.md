# Problem 6: Merkle Tree Construction and Proof Verification

## Result: VERIFIED (2 attempts)

## Approach

Three methods verified: `ComputeLayerImpl`, `BuildMerkleTree`, and `ExtractProof`.

### ComputeLayerImpl

Defined an auxiliary function `IterResult(layer, i)` representing what the iterative loop should produce from index `i` onward. The loop invariant is: `result + IterResult(layer, i) == IterResult(layer, 0)`.

Then proved `IterResult(layer, 0) == ComputeNextLayer(layer)` by structural induction, handling:
- Even-length layers: `ComputeNextLayer` recurses on `layer[2..]`
- Odd-length layers: `ComputeNextLayer` calls `ComputeNextLayerOdd` on `layer[2..]`

Used separate lemmas `IterResultMatchesRecursive` and `IterResultMatchesOdd` to handle the two recursive function variants.

### BuildMerkleTree

Leaf hashing loop: straightforward index-based invariant.

Layer-building loop invariants:
- `tree[|tree|-1] == currentLayer` (last layer is current)
- Layer-to-layer relationship maintained for all existing layers
- `decreases |currentLayer|` (proved via `ComputeNextLayerLength`)

### ExtractProof

This was the most intricate verification. The key invariant is:
```
ComputeProofRoot(tree[0][leafIdx], proof, directions) == tree[level][idx]
```
At each level, the hash computed by following the proof so far equals the actual tree node at position `idx`.

Used two key lemmas:
1. **`ComputeProofRootAppend`**: Unfolds `ComputeProofRoot` when appending one proof step. Proved by induction on proof length.
2. **`ComputeNextLayerIndex`**: At position `pairIdx` in the next layer, the value equals `HashCombine(layer[2*pairIdx], layer[2*pairIdx+1])` (or `HashCombine(layer[2*pairIdx], 0)` for odd last element). Proved by induction on the layer, handling both `ComputeNextLayer` and `ComputeNextLayerOdd` recursions.

The proof connects: appending a sibling hash to the proof produces `tree[level+1][idx/2]`, using the tree structure to link layers.
