## Approach

Added a helper lemma `SumRangeAppend` relating `SumRange(s, lo, hi+1)` to `SumRange(s, lo, hi) + s[hi]` (i.e., extending the sum by one element). This is the key inductive step that connects the running sum to the recursive SumRange function.

### Outer loop invariants:
- `|output| == i` (tracks how many rows processed)
- `forall r :: 0 <= r < i ==> |output[r]| == |input[r]|` (row lengths match)
- `forall r, c :: 0 <= r < i && 0 <= c < |input[r]| ==> output[r][c] == SumRange(input[r], 0, c + 1)` (prefix sum correctness for completed rows)

### Inner loop invariants:
- `|prefixRow| == j` (tracks progress)
- `runningSum == SumRange(row, 0, j)` (running sum is partial SumRange)
- `forall c :: 0 <= c < j ==> prefixRow[c] == SumRange(row, 0, c + 1)` (completed entries correct)

The `SumRangeAppend` lemma is called at the start of each inner iteration to establish `SumRange(row, 0, j+1) == SumRange(row, 0, j) + row[j]`.
