# Problem 3: BFS Reachability

## Result: VERIFIED (10 attempts)

## Approach

This was the hardest problem that was fully verified. Required 10 attempts.

### Key Design Decisions

1. **Ghost paths map**: Maps each visited node to a concrete path from source, witnessing reachability (soundness direction).

2. **Ghost expanded set**: Tracks which visited nodes have been fully processed (all neighbors explored).

3. **Frontier invariant**: "Every expanded node has all its neighbors in visited." This, combined with "all visited nodes are expanded when queue is empty," proves completeness.

4. **No-duplicates invariant**: `forall i, j :: 0 <= i < j < |queue| ==> queue[i] != queue[j]`. This was critical for proving that after adding `current` to `expanded`, `current` is no longer in the multiset of the queue.

5. **Helper lemma `ClosedContainsReachable`**: If a set containing source is closed under edges (all neighbors of members are also members), then it contains all reachable nodes. Proved by induction on paths.

### Errors Encountered (across 10 attempts)

1. **Attempts 1-2**: Frontier invariant `forall u in visited, all neighbors in visited or queue` failed on entry because source's neighbors aren't yet explored.
2. **Attempt 3**: `ghost-context continue statement is not allowed` - Dafny doesn't allow ghost-controlled flow.
3. **Attempts 4-5**: `SetOf(queue)` approach had termination issues; `visited - SetOf(queue)` isn't a well-founded decreasing measure.
4. **Attempts 6-7**: `multiset(queue)` tracking failed because Dafny couldn't prove the invariant was maintained across the outer loop body when adding `current` to `expanded`.
5. **Attempt 8**: Added explicit `expanded` ghost set with frontier only for expanded nodes. Multiset invariant still failed.
6. **Attempt 9**: Added `current !in multiset(queue)` to inner loop invariant. Trigger warning on `GraphNodes`.
7. **Attempt 10**: Fixed trigger warning. Verified.

### Critical Insight
The no-duplicates invariant on the queue was essential. Without it, Dafny cannot verify that after dequeuing `current` and adding it to `expanded`, `current` no longer appears in the queue (which would violate the `multiset(queue) ==> !in expanded` invariant).
