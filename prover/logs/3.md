# Problem 3: Sequence Slicing

## Status: VERIFIED

## Attempts: 1

## Strategy

The function `countChar` recurses from the front (`s[1..]`), and the loop also processes from the front. The difficulty is that slicing operations (`s[1..]`) create new sequences, and Dafny struggles to relate `countChar(s[..i], c)` to `countChar(s[..i+1], c)` without help.

### Attempt 1 (SUCCESS)

**Invariant added:**
- `0 <= i <= |s|`
- `count == countChar(s[..i], c)`

**Lemma added:**
- `countCharAppend(s, x, c)`: proves `countChar(s + [x], c) == countChar(s, c) + (if x == c then 1 else 0)`
  - Recursive proof: base case when `|s| == 0`, inductive step uses `t[1..] == s[1..] + [x]`.

**Key assertions:**
- `s[..i+1] == s[..i] + [s[i]]` (slice decomposition)
- `s[..i] == s` at the end

**Why it works:** Similar to Problem 1, the function definition recurses one way (from the front via `s[1..]`) but we need to reason about appending elements. The `countCharAppend` lemma establishes how appending an element changes the count, which is the exactly what the loop body does at each step. The slice assertion `s[..i+1] == s[..i] + [s[i]]` connects the loop's index-based view to the lemma's append-based view.
