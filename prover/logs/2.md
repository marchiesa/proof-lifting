# Problem 2: Trigger Gap on Existential

## Status: VERIFIED

## Attempts: 1

## Strategy

The `HasPair` predicate uses a specific trigger annotation `{:trigger s[i], s[j]}` on the existential. The challenge is that the negation of `HasPair` requires a universal statement (no pair sums to target), and the outer loop invariant must match the trigger pattern.

### Attempt 1 (SUCCESS)

**Outer loop invariant:**
- `0 <= i <= |nums|`
- `!found`
- `forall a, b {:trigger nums[a], nums[b]} :: 0 <= a < i && a < b < |nums| ==> nums[a] + nums[b] != target`

**Inner loop invariant:**
- `i + 1 <= j <= |nums|`
- `forall b {:trigger nums[b]} :: i + 1 <= b < j ==> nums[i] + nums[b] != target`

**Key assertions in the found case:**
- `nums[i] + nums[j] == target`
- `0 <= i < j < |nums|`

**Why it works:** The outer loop invariant uses the same trigger `{:trigger nums[a], nums[b]}` as the `HasPair` predicate. This ensures that Dafny can connect the negation of the universal invariant to `!HasPair`. When `found` is set to true, the concrete values of `i` and `j` directly witness the existential in `HasPair`. When the method falls through without finding a pair, the accumulated universal invariant exactly matches the negation of `HasPair`.
