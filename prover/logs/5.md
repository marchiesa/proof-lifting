## Strategy

Three postconditions require relating the frequency map to `SeqToMultiset(nums)`:
1. Every entry in `freq` has the correct count
2. Every element with positive multiplicity in the multiset is in `freq`
3. Elements not in the multiset (or with zero multiplicity) are not in `freq`

The function `SeqToMultiset` recurses from the front. The loop builds the map front-to-back.

### Attempt 1 (SUCCESS)

**Invariants:**
- `0 <= i <= |nums|`
- `forall v :: v in freq ==> freq[v] == SeqToMultiset(nums[..i])[v]`
- `forall v :: v in SeqToMultiset(nums[..i]) && SeqToMultiset(nums[..i])[v] > 0 ==> v in freq`
- `forall v :: v !in SeqToMultiset(nums[..i]) || SeqToMultiset(nums[..i])[v] == 0 ==> v !in freq`

**Lemma:**
- `SeqToMultisetAppend(s, x)`: proves `SeqToMultiset(s + [x]) == SeqToMultiset(s) + multiset{x}`
  - Uses a calc block with explicit multiset algebra steps.

**Key assertions:**
- `nums[..i+1] == nums[..i] + [nums[i]]`
- `nums[..i] == nums` at the end

**Why it works:** The invariants mirror the three postconditions exactly, just using `nums[..i]` instead of `nums`. The `SeqToMultisetAppend` lemma tells Dafny how adding one element changes the multiset. Since multiset operations (`+`, membership, multiplicity) are well-supported by Z3, the invariant maintenance after the map update is straightforward once Dafny knows the multiset decomposition. The calc block in the lemma provides an explicit chain of equalities that Dafny can verify step by step.
