# Problem 4: Non-linear Arithmetic

## Status: VERIFIED

## Attempts: 1

## Strategy

The postcondition `count == rows * cols` involves non-linear arithmetic (multiplication), which Z3 often struggles with. The nested loop requires invariants that express the count in terms of the loop indices.

### Attempt 1 (SUCCESS)

**Outer loop invariants:**
- `0 <= i <= rows`
- `count == i * cols`

**Inner loop invariants:**
- `0 <= j <= cols`
- `count == i * cols + j`

**Why it works:** The invariants decompose the non-linear relationship into a form Z3 can handle. After the inner loop, `j == cols` so `count == i * cols + cols == (i+1) * cols`, which is what the outer loop needs after incrementing `i`. The multiplication `i * cols` is non-linear but Z3 can handle it when the relationship is stated directly as an invariant rather than being inferred. The key insight is that stating `count == i * cols + j` in the inner loop avoids requiring Z3 to reason about multiplication across loop boundaries.
