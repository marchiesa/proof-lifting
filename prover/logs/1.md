# Problem 1: Recursive Sum with Fuel Trap

## Status: VERIFIED

## Attempts: 1

## Strategy

The function `sumSeq` recurses from the end of the sequence (`s[..|s|-1]`), but the loop iterates from the front. The key difficulty is the mismatch between how the function decomposes the sequence and how the loop processes it.

### Attempt 1 (SUCCESS)

**Invariant added:**
- `0 <= i <= |nums|`
- `total == sumSeq(nums[..i])`

**Lemma added:**
- `sumSeqAppend(s, x)`: proves `sumSeq(s + [x]) == sumSeq(s) + x`
  - This bridges the gap between the end-recursive definition and the front-to-back loop.

**Key assertions:**
- `nums[..i+1] == nums[..i] + [nums[i]]` (slice decomposition)
- `nums[..i] == nums` at the end (when `i == |nums|`)

**Why it works:** The lemma `sumSeqAppend` gives Dafny the "fuel" it needs to unfold the recursive definition in the right direction. Without it, Dafny tries to unfold `sumSeq(nums[..i])` from the end, but the loop adds one element at a time from the front. The lemma proves the connection once and for all, so the invariant maintenance becomes straightforward.
