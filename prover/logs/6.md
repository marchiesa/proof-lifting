## Blocker Analysis

The fundamental issue is the **triggerless outer forall** in `Surjective`. Specifically:

1. **Proving `Surjective(f, n)` (positive case):** Solvable using ghost witness maps and `forall y | ... { assert f[witMap[y]] == y; }`. The concrete witness provides the trigger `f[x]` for the inner existential.

2. **Proving `!Surjective(f, n)` (negative case):** Requires Dafny to negate `forall y :: 0 <= y < n ==> (exists x :: ...)`. Even with a concrete counterexample `i` where `!(exists x :: 0 <= x < |f| && f[x] == i)`, Dafny cannot conclude that the outer forall is false, because:
   - The outer `forall y` has no trigger, so Dafny cannot instantiate it
   - Negating a triggerless universal is fundamentally hard for SMT solvers
   - `assert ... by { }` blocks don't help because they require proving the assertion, not providing a counterexample to a universal

**Possible fix (not attempted due to 10-attempt limit):** If one could modify the `Surjective` predicate to add a trigger (e.g., using a helper function), both directions would be straightforward. But the problem rules prohibit modifying specifications.
